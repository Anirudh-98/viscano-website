Motion is a core UX and brand tool, not visual garnish. How to use animation with intent — and when to hold back.

## The Decoration Problem

Most brands add animation for one reason: it looks impressive in demos. The result is interfaces that feel sluggish, websites where everything flies in from the left, and brand films that are all kinetic energy and no narrative.

Animation used as decoration costs users attention without giving them anything in return. It's a tax on experience.

## What Motion Actually Does

When used with intent, motion does specific, functional things:

**1. It communicates state transitions**  
When a button animates on press, the user knows their input was received. When a modal slides in, the user understands that new context has arrived. Motion makes system state legible.

**2. It establishes spatial relationships**  
When a panel slides in from the right, the user understands that it came from the right — and that dismissing it will send it back. Motion creates a mental map of the interface.

**3. It directs attention**  
A subtle pulse draws the eye. A number counting up holds it. Used deliberately, motion can guide a user through the exact sequence of attention you've designed for.

**4. It expresses brand personality**  
The way your interface moves is part of your brand voice. A luxury brand moves slowly, deliberately. A consumer app moves quickly, bouncily. Motion is a brand tone setting.

## The Three Questions Before Any Animation

Before adding any motion to an interface, ask:

1. **What is this communicating?** If you can't answer this, it's decoration.
2. **Does it add time to the user's task?** If yes, it needs a strong justification.
3. **Is it appropriate to the context?** A finance app and a social app have different motion languages.

## Principles for Motion That Works

**Use easing, not linear timing.** Linear animation looks mechanical. Ease-in-out looks physical. The `[0.16, 1, 0.3, 1]` cubic-bezier used extensively in Viscano's own UI creates a spring-like feel that reads as natural.

**Keep durations short.** Most UI animations should be 150ms–400ms. Anything over 500ms starts to feel slow. The exception is hero/entrance animations where the animation is part of the content.

**Establish a motion system.** Don't animate ad hoc. Define a set of durations and easings that represent your brand — fast, medium, slow — and use them consistently.

**Animate meaning, not decoration.** If removing the animation doesn't change what the user understands, remove it.

## Brand Films Are Different

In a brand film context, motion is content — not interface. Here the rules shift. Slow, deliberate camera moves communicate confidence. Fast cuts communicate energy. 3D animation can communicate technology and precision. The question isn't "how fast?" — it's "what do we want the audience to feel?"
